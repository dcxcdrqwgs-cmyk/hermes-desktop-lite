# 安利一个 Hermes Agent 的第三方桌面客户端

> 把本地 AI Agent 变成真正「可用」的桌面应用

---

## 前言：我又「重复造轮子」了

是的，我又造轮子了。

但这次不太一样——我不是要再造一个「更强大」的 Hermes，而是想做一个**更「像应用」的 Hermes**。

Hermes Agent 本身能力很强：本地跑模型、管理记忆、执行文件、调用工具……但它主要是 **CLI 命令行 + Web 管理后台**的形态。

这对于开发者来说很爽，但对于「想把它当作日常 AI 助手用」的人来说，**总觉得缺了点什么**。

缺的是一个**真正放在 Dock 栏、点开就能聊、随时能查文件、像普通软件一样使用**的桌面客户端。

所以，我做了 **Hermes Desktop Lite**——一个专注于「日常使用」的极简桌面客户端。

---

## 它是什么？

**Hermes Desktop Lite** 是一个基于 Tauri 2 + React 19 构建的第三方桌面客户端，目标只有一个：

**让 Hermes Agent 变成一个你每天都会点开的「应用」**，而不是一个需要特意打开的管理后台。

## 🎯 核心设计哲学

### 1️⃣ 极简 & 够用就好

很多功能你可能**几个月都不会点一次**（如 Skills 市场、复杂 Cron 配置）。  
这个客户端只保留**每天都会用的 7 个功能**，砍掉干扰，专注高频场景。

### 2️⃣ 不重复造轮子

所有 AI 能力、工具执行，全部走 **Hermes Gateway API**。  
我只是做了一个**外壳 + UI 层 + 本地文件系统桥接**。  
它是 Hermes 的「皮肤」，内核还是那个强大的本地 Agent。

### 3️⃣ 双模式开发

- **Browser 模式**：`npm run dev` - 纯前端，快速调试 UI
- **Tauri 模式**：`npm run tauri dev` - 桌面应用，完整功能

改样式调交互，不需要启动 Rust 环境。效率优先。

## 为什么选它？3 个核心理由

### ✅ 理由 1：它是「应用」，不是「控制台」

Hermes Web Admin 是**管理后台**，功能全但打开成本高。  
Desktop Lite 是**日常应用**——放在 Dock 栏，点开即用，侧边栏常驻切换无感。

**体验差异**：就像「用 Slack 客户端」vs「用 Slack 网页版」——前者更像工具，后者只是网页。

---

### ✅ 理由 2：本地运行，数据完全自主

- 对话记录存你本地的 SQLite
- 配置文件在你 home 目录
- 文件操作直接读写本地磁盘
- **没有云服务，没有数据上传，没有隐私担忧**

---

### ✅ 理由 3：Tauri 架构，轻量原生

| 对比项 | Tauri | Electron |
|--------|-------|----------|
| 打包体积 | ~10MB | ~100MB+ |
| 内存占用 | 低 | 高 |
| 安全性 | Rust 后端，权限可控 | Node.js 全权限 |

轻量、快速、系统原生体验。

## 如何使用？

### 方式一：浏览器模式（30 秒体验）

不需要装 Rust，直接用浏览器试试 UI：

```bash
git clone https://github.com/your-username/hermes-desktop-lite.git
cd hermes-desktop-lite
npm install
npm run dev
```

浏览器打开 `http://localhost:5173` 即可。

> ⚠️ 浏览器模式下文件编辑和终端功能不可用，建议用方式二完整体验。

---

### 方式二：桌面模式（完整体验）

```bash
npm run tauri dev
```

自动打开桌面窗口，所有功能可用。

**前提**：
- Node.js ≥ 20
- Rust 环境
- 本地已运行 Hermes Agent（默认端口 8642）

## 常见问题（FAQ）

**Q：这是官方的吗？**  
A：不是。这是第三方社区项目，与 Hermes 官方无直接关系。

**Q：数据安全吗？**  
A：所有数据都在你本地，不会上传云端。完全可控。

**Q：支持 Windows/Linux 吗？**  
A：支持。Tauri 跨平台，代码已适配，构建脚本包含各平台。

**Q：会和 Hermes 官方功能冲突吗？**  
A：不会。它只是 Hermes 的一个「前端壳子」，所有能力走 Gateway API。

**Q：模型配置在哪里？**  
A：模型配置（API Key、Base URL 等）通过 Hermes 环境变量管理，客户端提供**模型选择下拉菜单**（聊天界面右上角），选择当前会话使用的模型。

**Q：有 Hermes 运行日志查看吗？**  
A：暂未实现。当前版本聚焦对话、文件、任务、终端等高频功能。
